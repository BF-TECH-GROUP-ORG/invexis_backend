📦 Products Management

Complete product lifecycle management with images, variations, and inventory

🚀 Key Features

• File upload support (images/videos) • QR/Barcode generation • SKU auto-generation • Bulk operations • Search & filtering

GET /v1/products
Retrieve all products with advanced filtering, pagination, and search capabilities. Supports price range, category, brand, and stock filters.
Parameter Type Required Description
page number optional Page number (default: 1)
limit number optional Items per page (default: 20)
search string optional Text search in product name/description
category string optional Filter by category ID
brand string optional Filter by brand name
minPrice number optional Minimum price filter
maxPrice number optional Maximum price filter
companyId string required Company identifier

GET /v1/products/{id}
Retrieve a specific product by its ID with complete details including pricing, inventory, and variations.
Parameter Type Required Description
id string required Product ID (MongoDB ObjectId)
PUT /v1/products/{id}
Update an existing product. Supports partial updates and file uploads for new images.
DELETE /v1/products/{id}
Soft delete a product. Automatically cleans up Cloudinary images, QR codes, and barcodes in the background.
POST /v1/products/bulk
Create multiple products in a single request. Supports batch processing with individual success/failure tracking.
GET /v1/products/search/product
Advanced product search with text matching and multiple filters.
GET /v1/products/lookup/{barcode}
Lookup product by barcode or QR code scan. Returns complete product information for POS integration.

📊 Stock Management

Inventory tracking, stock adjustments, and movement history

POST /v1/stock/in
Add inventory to stock (restocking). Creates stock change records for audit trail.

📥 Request Body:

{
  "productId": "507f1f77bcf86cd799439011",
  "quantity": 50,

  "cost": 2500.00,

  "location": "Warehouse A"

POST /v1/stock/out
Remove inventory from stock (sales, damage, returns). Validates available quantity.
POST /v1/stock/lookup
Lookup product information by QR code or barcode scan for POS operations.
GET /v1/stock/history
Get comprehensive stock movement history with filtering options.

🛠️ Integration Notes

Authentication: Most endpoints require JWT token in Authorization header

File Uploads: Use multipart/form-data for endpoints with file support

Error Handling: All endpoints return consistent error format with success boolean

Rate Limiting: API includes rate limiting - check rate limit headers

Caching: GET endpoints support ETag caching for performance
